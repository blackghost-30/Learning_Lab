# 第二节课：C++简介

## 1.C++简介

- C++是一种**静态类型的、编译式的、通用的、大小写敏感的、不规则的**编程语言，支持**过程化编程、面向对象编程和泛型编程**：
  - 静态类型的：即在定义变量时需要指定变量的类型，且不允许后续更改变量的类型；
  - 编译式的：需对源文件进行编译后才能运行，整个源文件都没有Bugger，与之相对的是解释式语言；
- C++被认为是一种**中级语言**，综合了高级语言和低级语言的特点，它与C语言是兼容的；
- C++由**Bjarne Stroustrup**于1979年在新泽西州美利山贝尔实验室开始设计开发的；
- C++进一步扩充和完善了C语言，最初命名为带类的C，后来在1983年更名为C++；
- 任何合法的C程序都是合法的C++程序；



## 2.面向对象程序设计

C++支持面向对象的程序设计，面向对象有如下四大特性：

- **封装**：
  - 封装是将**数据和方法组合在一起**，对外部隐藏实现细节，只公开**对外提供的接口**；
  - 封装可以提高安全性、可靠性和灵活性；
- **继承**：
  - 继承是**从已有类中派生出新类**，新类具有已有类的属性和方法，并且可以扩展或修改这些属性和方法；
  - 继承可以提高代码的复用性和可扩展性；
- **多态**：
  - 多态是指**同一种操作作用于不同的对象，可以有不同的解释和实现**；
  - 多态可以通过接口或继承实现，可以提高代码的灵活性和可读性；
- **抽象**：
  - 抽象是**从具体的实例中提取共同的特征，形成抽象类或接口，以便于代码的复用和扩展**；
  - 抽象类和接口可以让程序员专注于高层次的设计和业务逻辑，而不必关注底层的实现细节；



## 3.C++标准

### 3.1 C++的标准组成

标准的C++由三个重要部分组成：

- **核心语言**：提供了所有构建块，包括变量、数据类型和常量等等，它规定了语言的语法；
- **C++标准库**：提供了大量的函数，用于操作文件、字符串；
- **标准模板库（STL）**：属于标准库，提供了大量的方法，用于操作数据结构等；

### 3.2 C/C++的标准库

- C标准库（C Standard Library）：
  - 基础IO：
    - <stdio.h>，主要是printf()和scanf()函数；
  - 内存管理：
    - <stdlib.h>，主要是malloc()和free()函数；
    - <string.h>，主要是memcpy()和memset()函数；
  - 数学运算：
    - <math.h>，主要有sin()、sqrt()、pow()等函数；
  - 时间与日期：
    - <time.h>，主要是获得时间戳；
  - 字符处理：
    - <ctype.h>，主要用于判断字符是字母还是数字；
  - 信号与异常：
    - <signal.h>，主要用于信号和异常的处理；
- C++标准库（C++ Standard Library）：
  - IO流：
    - <iostream>，主要是cin/cout等函数；
    - <fstream>，主要是文件读写；
    - <sstream>，负责字符串流；
  - 字符串：
    - <string>，主要有std::string类；
  - 容器（STL核心）：
    - 序列容器：<vector>、<list>、<deque>、<array>；
    - 关联容器：<map>、<set>、<unordered_map>、<unordered_set>；
  - 算法：
    - <algorithm>，主要包括sort()、find()、count()等函数；
  - 智能指针与内存：
    - <memory>，主要包括unique_ptr()、shared_ptr()等函数；
  - 多线程：
    - 包括了<thread>、<mutex>、<condition_variable>等文件；
  - 文件系统：
    - <filesystem>文件；
  - 类型工具
    - <type_traits>，用于模板元编程；

- 无论是C还是C++，**它们的这些标准库都是由编译器提供的**，所以可以在源文件中直接#include，而不像STM32中的ST官方库一样需要复制到工程下；



## 4.ANSI标准

- ANSI标准是为了确保C++的便携性，所编写的代码在Mac、UNIX、Windows、Alpha计算机上都能通过编译；
- 所有主要的C++编译器的制造商都支持ANSI标准
